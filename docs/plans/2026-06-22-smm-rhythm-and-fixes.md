# Content Factory — SMM-ритм + фиксы постинга (план новой сессии)

> Создан 2026-06-22 по итогам боевого теста дайджеста Юрием. Предыдущая сессия: 4 блока (анти-повтор дайджеста, загрузка видео 500МБ, ERP booking-links, модернизация редактора Stories) — в проде. Этот план = следующий шаг: довести постинг до боевого ритма.

## Context (зачем)
Юрий впервые постил из дайджеста (прямая публикация, без редактора). Что всплыло:
1. **Дубль хэштегов** в VK И IG постах (баг).
2. **VK-сторис без живой кнопки** «Забронировать» (прямой путь из дайджеста не передаёт ссылку).
3. Дайджест даёт **кучу в день** (2 фото + 1 сторис) без ритма.
Решение Юрия по ритму: **3 сторис/день (утро/день/вечер) + лента 5-6/нед из реального материала туров** (Reels в приоритете, выходные легче, «нет сильного кадра — пропускаем»). Плюс новая идея — **слот-филл** (промоутить туры, где УЖЕ есть люди → максимальная загрузка одного слота) и **доска вдохновения IG**.

## Блок A — Баг: дубль хэштегов (быстрый, высокий приоритет)
**Корень (подтверждён на реальных постах):** AI вписывает хэштеги ИНЛАЙН в текст (`...вода\n\n#НаWоде #SUP...`), а система отдельно хранит `version.hashtags`, и паблишер клеит массив в конец ещё раз → два набора (в тексте одни теги, в массиве другие). Видно в `publish-runner.ts` (`hashtags: version.hashtags`) + паблишеры `vk.ts:62`/`postmypost.ts:133`/`telegram.ts:25` (`${text}\n\n${hashtags.map('#'+h)}`).
**Фикс:** хэштеги — в ОДНОМ месте (массив = источник правды). Вырезать инлайн-теги из AI-текста при сохранении:
- `services/daily-digest.ts`: после копирайтера (`copy.text` ~стр.383) и в `adaptForPlatforms` (~стр.313-339) применять `stripInlineHashtags(text)` (util: убрать токены `#\S+` + лишние пустые строки в конце) перед записью в `proposedText`/`adaptations[].text`.
- **IG кап 5 тегов** (IG с 12.2025 ограничил до 5/пост): в `adaptForPlatforms` для INSTAGRAM — `hashtags.slice(0,5)`. VK можно оставить ~10.
- Защитно (не обязательно): в `publish-runner.ts` перед публикацией прогнать `baseText` через `stripInlineHashtags` (единый чокпоинт), чтобы и ручные посты не дублировали.
**Проверка:** опубликовать тест-пост → один набор хэштегов; IG ≤5.

## Блок B — Баг: живая VK-кнопка в сторис из дайджеста (быстрый)
**Корень:** `routes/auto-post.ts` `approve-publish` (~стр.141) шлёт `storiesOptions = { skipOverlay }` — БЕЗ `linkText/linkUrl`. В редакторе кнопка есть, прямой путь её теряет.
**Фикс:** для VK + STORIES при прямой публикации авто-подставлять «Бронь ВК Сторис»:
- В `approve-publish` перед публикацией версии: если `platform==='VK' && postType==='STORIES'` → подтянуть ссылку через `getBookingLinks()` (scope содержит `story`+`vk`; fallback `vk`) → `storiesOptions.linkText='book'`, `storiesOptions.linkUrl=<url>`. UTM добавится в `applyUtmForPublish`.
- Опционально хранить выбранную ссылку на `AutoPostTask` (новое поле) + дать выбор в модалке `DigestView`. Минимум — авто-дефолт на бэке.
**Проверка:** опубликовать сторис из дайджеста в VK → нативная кнопка «Забронировать» с ref-ссылкой.

## Блок C — Ритм-движок дайджеста (основное)
Сейчас `generateDigestForBusiness` тупо просит `count:3` идей (PHOTO/STORIES). Нужен ритм.
**Целевой ритм (решение Юрия):**
- **3 сторис/день с ролями:** утро = погода+слоты+кнопка брони; день = слот-филл (Блок D) + «прямо сейчас на воде»; вечер = рекап дня + «на завтра осталось N мест».
- **Лента 5-6/нед** (будни активнее, выходные легче), ротация 10 рубрик, Reels в приоритете. Правило «нет сильного материала — пропустить».
**Реализация:**
- AppConfig: `digest_story_roles` (вкл/выкл утро/день/вечер), `digest_feed_days` (дни недели для ленты), `digest_feed_rubric_rotation` (порядок рубрик). Дефолты разумные.
- `daily-digest.ts`: вместо одного `count:3` — генерировать (a) утреннюю сторис всегда, (b) день/вечер сторис по ролям, (c) ленту только в feed-день с рубрикой дня (учитывая `recentProposals` анти-повтор — уже есть). Время публикации подсказывать (или авто-schedule через `schedulePostVersion`).
- `prompt-builder.ts`: отдельные промпты под роли сторис (утро/день/вечер) — разный вход (погода/слоты/рекап).
- 10 рубрик: `services/ai/nawode-strategy.ts` `NAWODE_RUBRICS`. Ротацию вести по `recentProposals` (какие рубрики давно не были).
**Решить:** дайджест предлагает все 3 сторис утром (Юрий публикует/планирует в течение дня) ИЛИ scheduler гоняет 3×/день. Рекомендация: предлагать пачкой утром + кнопка «Запланировать на 12:00/19:00».

## Блок D — Слот-филл сторис (фича, ради денег)
Промоутить туры с УЖЕ записанными людьми → max загрузка слота + social proof.
**Данные:** новый `getHotSlots(daysAhead)` в `nawode-data.ts` — читать `service_slots` (вместимость) + `bookings` (записано) → `{date,time,tourName,booked,capacity,remaining,productId}`, фильтр «есть записи, но есть места» (0 < booked < capacity), сорт по «почти заполнен». Паттерн ERP-чтения — как `getBookingRoiByRef` (CTE, settled-payment, ленивый `import('bun')`). Сверить имена таблиц/колонок через MCP `nawode-erp-db` (`service_slots`, `bookings`, `products`/`product_waypoints` для названия тура).
**Использование:** дневная сторис берёт «горячий» слот → «Завтра закатный тур, уже N человек — добивайте до M, поедем точно!» + кнопка брони. Дизайн через satori (`buildStoryDesign`).

## Блок E — Конкуренты VK + доска вдохновения IG
- **VK:** мониторинг есть (`competitor-poller.ts`, `CompetitorAccount`/`CompetitorPost`, «залетевшие» → в дайджест). Расширить список пабликов (`seed-competitors.ts` или UI) — Юрий накидает.
- **IG:** автомониторинг НЕЛЬЗЯ (нет публичного API, скрейпинг против ToS, серая зона, Meta-экстремизм). Вместо этого — **доска вдохновения:** Юрий кидает ссылку на зашедший рилс → AI разбирает формулу (хук/формат/идея) → в банк идей (новый `source='inspiration'` у AutoPostTask ИЛИ Idea). + периодический тренд-ресёрч (web search свежих SUP-форматов) раз в 2-4 нед.
- Аккаунты-референсы (Юрий назвал): `sup4you.ru`, `sup_fitness__`, `progress.ride`, `friends.sup` — попросить ссылки на конкретные рилсы для разбора (web-поиск RU IG-профили видит плохо).

## Тренды (ресёрч 2026, для промптов/рубрик)
- **Серийные рубрики** (рилс-сериал) — топ-инструмент удержания (наши 10 рубрик → сериалы).
- Хук в 1-3 сек (3-сек hold-rate >60% = ×5-10 охвата), 30-60 сек, трендовые аудио ловить на росте.
- Форматы: POV/день-из-жизни, «эстетик-суперкат» (закаты/замок), лайфхаки (техника, «с детьми?»), UGC (брендовый хэштег + репост клиентов), опросы.
- IG: **5 хэштегов макс** (с 12.2025). Reels ~2× охвата.

## Порядок и верификация
1. A (хэштеги) + B (VK-кнопка) — быстрые, делаем первыми, деплой, проверить публикацию.
2. D (getHotSlots) — основа для дневной сторис.
3. C (ритм-движок) — крупное, опирается на A/B/D.
4. E (конкуренты/вдохновение) — последним.
- Верификация: `bun run test` (≥185), `bun run build`, Playwright на проде после деплоя (паттерн прошлой сессии: создать тест-данные через `docker exec content-factory-backend-1 bun -e`, проверить, убрать). Деплой — только по команде Юрия.
