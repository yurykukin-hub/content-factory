### A1 — Отложка больше не теряет кнопку ВК и музыку
- Раньше: при планировании сторис кнопка ВК (`linkText/linkUrl`) **молча терялась** (scheduler хардкодил `skipOverlay:true`). Это был реальный баг.
- Теперь: новое поле `PostVersion.publishOptions` (миграция, additive). `/schedule` принимает и сохраняет опции (Zod). Scheduler читает их при публикации. StoryEditor шлёт кнопку.

### A2 — Управление запланированными сторис
- Раньше: запланированный сторис **блокировался как опубликованный** (нельзя отредактировать).
- Теперь: статусы разделены (`isPublished` только для PUBLISHED, `isScheduled` отдельно). Запланированный сторис **редактируется и перепланируется**. `post.status` теперь корректно показывает SCHEDULED → работает фильтр «Запланированные». В списке Stories видно **время публикации + кнопка отмены**.

### C — Утренний AI-агент (вау-фактор)
- Каждое утро (07:00 МСК) агент читает **реальную погоду НаWоде** (прямой read-only доступ к nawode PostgreSQL через `Bun.sql`) + бронирования + контент-план (10 рубрик, сезонный подход) + недавние посты → **Sonnet** → 3-4 предложения: *формат, канал, готовый текст, идея визуала, обоснование*.
- Новый раздел в интерфейсе: **«Дайджест»** (sidebar). Карточки предложений с кнопками **«Одобрить → черновик»** и «Отклонить».
- Одобрение создаёт **черновик поста** (human-in-the-loop, без авто-публикации) → виден в интерфейсе → дорабатывается вручную → публикуется/планируется существующим флоу.
- **Проверено вживую:** агент сгенерировал 4 предложения под реальную погоду (чт +24°C, ветер 4 м/с → сторис «время SUP», пост про Выборгский замок с привязкой к погоде, рилс «первый раз на доске», знакомство с командой). Все — только VK, с ссылкой nawode.ru, по закону РФ (без пиара Instagram).
- Telegram-доставка с кнопками **готова в коде** (graceful) — включится, когда добавишь токен бота (см. §4).

### A3 — Дизайн-пресеты сторис
- 5 готовых пресетов оформления текста в один клик (Чистый низ, Крупный заголовок, Минимал, Акция, Светлый верх). Глобальные (видны всем проектам). Расширены поля: выравнивание текста + форма фона.

### A4 — Музыка в видео-сторис
- В редакторе видео-сториса — выбор трека из **Звуковой студии**. Трек **вшивается в видео** (ffmpeg, «bake once») и переиспользуется для всех каналов. Привязка сохраняется при отложке.
- Почему вшиваем: нативную музыку в сторис VK/IG через API добавить нельзя.

## 2. 🔍 Как проверить утром (по шагам)

1. **Утренний агент:** открой https://content.yurykukin.ru → в меню слева **«Дайджест»**. Увидишь 4 готовых предложения на сегодня (погода-aware). Нажми **«Сгенерировать сейчас»** чтобы перегенерировать. Нажми **«Одобрить → черновик»** на любом — создастся черновик и откроется редактор: доработай и опубликуй/запланируй.
2. **Отложка с кнопкой ВК:** «Stories» → открой/создай сторис → задай кнопку (напр. «Забронировать» → nawode.ru) → «Предпросмотр и публикация» → выбери дату → «Запланировать». Раньше кнопка терялась — теперь сохранится. В списке Stories у запланированного видно **время + значок отмены**.
3. **Пресеты:** открой любой сторис → блок «Шаблоны» (вверху настроек) → 5 пресетов 🌊🔆✦🔥☀️ — клик применяет оформление.
4. **Музыка:** открой видео-сторис (где медиа — видео) → блок «Музыка» (если в Звуковой студии есть готовые треки НаWоде) → выбери трек → опубликуй. Музыка будет в видео.
5. **Все форматы (B1/B3):** меню «Контент» (бывш. Stories) теперь показывает посты ВСЕХ типов с фильтром (Все/Сторис/Посты/Фото/Видео/Reels). Обычный пост (не сторис) откроется в редакторе постов, где можно адаптировать под платформы, **опубликовать ИЛИ запланировать** (datetime + «В план»).

## 4. ⚠️ Решения, где я выбрал дефолт (нужно твоё слово)

1. **Данные агента — прямой read-only доступ к БД НаWоде** (`NAWODE_DATABASE_URL` в `.env.prod`), а не HTTP-эндпоинт `daily-summary`. Причина: эндпоинт существует, но под авторизацией на другом VPS, а у НаWоде в CF не заданы `erp_base_url/erp_api_key`. Прямой доступ надёжнее и не требует трогать прод НаWоде ночью. Инкапсулирован в `nawode-data.ts` — легко заменить на HTTP позже. *Обсудить: оставляем так или переходим на эндпоинт?*
2. **Я включил ежедневный агент** (`digest_enabled=true`, `digest_time_utc=04:00 UTC = 07:00 МСК`). Он будет создавать предложения каждое утро (без авто-публикации). **Выключить:** в БД `UPDATE app_config SET value='false' WHERE key='digest_enabled';` или через Settings.
3. **Telegram-доставка агента — graceful (пока выключена):** нужен бот. Создай бота у @BotFather и добавь в `app_config`: `telegram_approval_bot_token` + `telegram_approval_chat_id` (твой chat_id). После этого утренние предложения будут приходить в Telegram с кнопками Одобрить/Отклонить. Вебхук ставится автоматически при старте (`setupTelegramWebhook`).
4. **WIP «AI-видео в сторис» закоммичен как baseline** (`73381bd`) — он был незакоммичен, typecheck зелёный. Если это была сырая работа — откати коммит.
