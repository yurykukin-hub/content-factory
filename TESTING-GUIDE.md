# 🧪 Как протестировать Content Factory + понять пайплайн

URL: **https://content.yurykukin.ru** · логин: **admin** · проект для тестов: **НаWоде**

---

## 0. Пайплайн целиком (как всё связано)

```
        СТРАТЕГИЯ                        ТАКТИКА (каждый день)              ТЫ (human-in-the-loop)
┌────────────────────────┐      ┌──────────────────────────────┐    ┌─────────────────────────┐
│  УМНЫЙ КОНТЕНТ-ПЛАН     │      │  УТРЕННИЙ АГЕНТ (07:00 МСК)    │    │  «Дайджест»             │
│  (скелет на месяц)      │      │  daily-digest.ts (Sonnet)     │    │  карточки предложений    │
│                         │      │                               │    │                         │
│  • 10 рубрик НаWоде     │─────▶│ читает: погоду+брони (ERP),    │───▶│ ✅ Одобрить → ЧЕРНОВИК   │
│  • события/праздники    │ план │ контент-план, недавние посты  │    │ ❌ Отклонить            │
│  • реальные брони       │ как  │ → 3–4 предложения НА СЕГОДНЯ  │    │                         │
│  • анти-повтор          │ контекст                              │    └───────────┬─────────────┘
└────────────────────────┘      └──────────────────────────────┘                │ создаётся Post (DRAFT)
                                                                                  ▼
                                                              ┌───────────────────────────────────┐
                                                              │  РЕДАКТОР (Stories / Posts)         │
                                                              │  дорабатываешь вручную → публикуешь │
                                                              │  ИЛИ планируешь (отложка)           │
                                                              └───────────────────────────────────┘
```

**Ключевая идея:** план = «о чём вообще писать в этом месяце» (стратегия). Агент = «что именно постим сегодня под реальную погоду/брони» (тактика). Ты остаёшься в контуре: ничего не публикуется само — агент только **предлагает**, ты одобряешь и контролируешь.

**Что делает пайплайн «умным»:**
- **Рубрики** — агент/план пишут в тоне бренда по 10 рубрикам НаWоде.
- **Реальные данные** — агент видит фактическую погоду и брони из ERP (не выдумывает).
- **Анти-повтор** — при генерации учитываются последние посты, чтобы не дублировать.
- **События** — праздники/поводы (День России, Пиратская гавань…) автоматически попадают в план на нужные даты.
- **Переделка** — любую ячейку плана можно переписать с твоим направлением.

## 1. 🌅 Утренний агент + human-in-the-loop (ГЛАВНОЕ — тестим в первую очередь)

Меню слева → **«Дайджест»**.

- [ ] Видишь **4 готовых предложения** на сегодня (Stories / Photo / Reels). У каждого: тип, канал, готовый текст, идея визуала, обоснование («почему сегодня»).
- [ ] Нажми **«Одобрить → черновик»** на любом → создаётся **черновик поста**, тебя перекидывает в редактор.
- [ ] В редакторе доработай текст/медиа → **«Опубликовать»** или запланируй (datetime + «В план»). ← это и есть автопубликация «через агента»: агент предложил → ты за 2 клика выпустил.
- [ ] Нажми **«Отклонить»** на другом → исчезает.
- [ ] Нажми **«Сгенерировать сейчас»** (вверху, admin) → агент сделает свежую пачку под текущую погоду.

> Каждое утро в **07:00 МСК** агент сам положит новые предложения в этот раздел (включено). Ничего не публикуется без твоего «Одобрить».

---

## 2. 🧠 Умный контент-план (Эпик D)

Меню → **«Контент-планы»**.

- [ ] Создай **AI-план** для НаWоде на период (напр. 01.06–30.06).
- [ ] Проверь, что в плане **есть рубрики** (Выборг с воды, Первый раз на доске…) и **события на точных датах**:
      - 01.06 «Открытие сезона», 12.06 «День России», 21.06 «Солнцестояние».
- [ ] У ячейки нажми **⟳ «переделать»** → впиши направление (напр. «сделай про детей») → AI перепишет тему/тип/описание.
- [ ] Нажми **AI-написать** (палочка) у ячейки → создаст пост; проверь, что **текст не повторяет** недавние посты (анти-повтор работает на истории).

---

## 3. 📅 Отложка сторис: кнопка ВК + управление (Эпик A1+A2)

Меню → **«Контент»** → фильтр **Сторис** → открой/создай сторис.

- [ ] Задай кнопку (напр. «Забронировать» → nawode.ru) → **«Предпросмотр и публикация»** → выбери дату → **«Запланировать»**.
- [ ] В списке у запланированной сторис видно **время + значок отмены** (📅✕).
- [ ] Открой запланированную → она **редактируется** (раньше блокировалась как опубликованная). Поменяй → снова «Предпросмотр» → перепланируй.
- [ ] Главное (это был баг): **кнопка ВК больше не теряется** при отложке — она сохранится к моменту публикации.

---

## 4. 🎨 Дизайн-пресеты сторис (Эпик A3)

В редакторе сторис, блок «Шаблоны» (вверху настроек):
- [ ] 5 пресетов: 🌊 Чистый низ · 🔆 Крупный заголовок · ✦ Минимал · 🔥 Акция · ☀️ Светлый верх — клик применяет оформление (позиция/цвет/фон/выравнивание).

## 5. 🎵 Музыка в видео-сторис (Эпик A4)

Открой сторис, где медиа = **видео**:
- [ ] Появляется блок **«Музыка»** (если в Звуковой студии есть готовые треки НаWоде) → выбери трек → опубликуй → музыка **вшита в видео**.
- [ ] Привязка сохраняется и при отложке.

## 6. 🎬 Все форматы (Эпик B)

Меню → **«Контент»**:
- [ ] Фильтр типов: Все / Сторис / Посты / Фото / Видео / Reels / Клипы — у каждой карточки бейдж типа.
- [ ] Создай пост → в редакторе **«Тип контента»** → выбери **Reels (IG)** или **Клипы (VK)** → подсказка формата + предупреждение «нужно видео».
- [ ] Привяжи видео → **«Опубликовать»** или запланируй (datetime + «В план»).
- [ ] Reels → IG (Postmypost type 4). Клипы → VK публикует как вертикальное видео (отдельного API «Клипов» у VK нет).

## 7. ⚙️ Автопубликация «дальше через агента» — как масштабировать

Сейчас (фаза human-in-the-loop): агент **предлагает** → ты одобряешь и публикуешь. Это сознательно — пока ты контролируешь качество.

Когда захочешь больше автоматизации:
1. **Telegram-кнопки** — добавь бота (`telegram_approval_bot_token` + `telegram_approval_chat_id` в Settings/AppConfig). Тогда предложения будут приходить в Telegram с кнопками «Одобрить/Отклонить» — одобряешь с телефона.
2. **Semi-auto** (следующая фаза) — одобрил → пост сам уходит в отложку на выбранное время.
3. **Full-auto** — для рутинных форматов (погода+слоты) публиковать без подтверждения.

Скажи, когда дойдём — это небольшие надстройки над тем, что уже есть.

---

## Что держать в голове при тестах
- **Дайджест включён** (07:00 МСК ежедневно), создаёт только черновики — без авто-публикации.
- **Telegram пока выключен** (нужен токен) — одобрение сейчас в вебе.
- **Брони в ERP пустые** (ранний сезон) — агент опирается на погоду + рубрики; когда появятся брони, он начнёт их использовать (анонсы/соцдоказательство).
- Если что-то странно — смотри **AI Логи** (стоимость/ошибки генераций).
